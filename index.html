LLDB 是什么?

LLDB 是一个下一代，高性能的调试器。它被构建为一个可复用的组件集，高度使用了规模更大的 LLVM 项目中已存在的库，比如 Clang 表达式解析器和 LLVM 反汇编器。

LLDB 是 Mac OS X 上 Xcode 默认的调试器，支持在桌面电脑、iSO 设备和模拟器上调试 C、Objective-C 和 C++ 。

LLDB 项目所有的代码，可在 LLVM 许可 标准下获取, 这是一个开源 "BSD-style" 许可证。

为什么是一个新的调试器?

为了达成我们的目标，我们决定从一个崭新的架构开始，将支持现代多线程程序，通过高效的手段处理调试符号，使用基于编译器的代码知识，拥有插件以支持功能和扩展。加之，我们希望这个调试器的功能对于其他的分析工具也是可用的，如脚本或编译后的程序，而无需要求这些工具都是 GPL 授权（暗指 GCC）。

编译器整合的好处

LLDB 当前将调试信息转为 clang 类型，所以它可以使用 clang 编译器基础设施。这允许 LLDB 支持表达式中最新的 C 、C++ 、Objective C 和 Objective C++ 语言特性和运行时，而无需重新实现任何相关功能。它同时利用这个编译器（clang），仔细处理所有 ABI 的细节，当为表达式生成函数调用时，反汇编指令时，和获取指令细节时，等等。

主要好处，包括：

  • 最新的 C 、C++ 、Objective C 语言支持
  • 多行表达式，可以声明局部变量和类型
  • 为表达式利用 JIT，当被支持时
  • 当 JIT 不能使用时，评估表达式的中间代码表示（IR）

可重用性

LLDB 调试器的 APIs 被作为一个 C++ 面向对象接口暴露，在一个共享库中。lldb 命令行工具工具连接，和使用这个开放 API。在 Mac OS X 这个共享库作为一个名为 LLDB.framework 的框架暴露，unix 系统通过 lldb.so 暴露。整个 API 还通过 Python 脚本绑定进行暴露，这允许 API 在 LLDB 嵌入式脚本解析器中使用，以及在加载了 lldb.py 模块的任何标准 Python 脚本文件。浏览 Python 参考 了解如何和何处 Python 可以通过 LLDB API 被使用相关的详细信息。

共享 LLDB API 不仅允许在调试中被使用 LLDB ，还可以在符号表现，反汇编，对象和符号文件反省（introspection）等地方使用。

平台支持

LLDB 目前支持以下平台, 但是针对新平台的移植也是很欢迎的：

  • Mac OS X i386 和 x86-64 桌面用户空间调试
  • i386 iOS 模拟器调试
  • ARM iOS 设备调试
  • Linux i386 和 x86-64 本地用户空间调试
  • FreeBSD i386 和 x86-64 本地用户空间调试
  • Windows i386 (*) 本地用户空间调试

(*) Windows 的支持正在开发当中。基础功能已经正常运作，伴随快速功能改进。

获取和参与

检出代码，使用：

  • svn co http://llvm.org/svn/llvm-project/lldb/trunk lldb

备注：LLDB 通常用最新的 trunk 构建：

  • Mac OS X 上用 Xcode
  • Linux 和 FreeBSD 上用 clang 和 libstdc++/libc++
  • NetBSD 上用 GCC 和 clang 以及 libstdc++/libc++
  • Windows 上用 VS 2012 或 更高版本使用 CMake

浏览 LLDB 构建页面 获取平台相关的构建指令。

讨论 LLDB 应当去 lldb-dev 邮件列表。提交 lldb SVN 模块的消息会自动发送到 lldb-commits 邮件列表，这也是补丁提交的首选邮件列表。
