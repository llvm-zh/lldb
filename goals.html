目标

开源调试器的现状是，能正常处理 C 应用的通用场景，但是不能恰当地处理很多“疑难问题”。比如，C++ 表达式解析，处理重载，模板，多线程和其他非凡的场景，都能应对一些基础的情况，但是并不可靠。

LLDB 的目标，是提供一个非常惊艳的“可运行”（just works）调试体验。我们计划解决那些令调试者困惑的长期存在的难题，所以你可以思考调试你的程序，而不是思考调试器的缺陷。

从长远来看，没有一个合适的理由去重新发明它自己的 C/C++ 解析器，类型系统，知道所有目标调用惯例细节，实现自己的反汇编器，等等。通过使用 LLVM 项目提供的已经存在的库，「我们相信其中的很多问题，都将得到很好的定义」，调试器可以专注于终于的问题，比如过程控制，高效的符号读取和索引，线程管理，和其他调试器相关的问题。

更多特殊目标，如下：

  • 构建库，为包括 IDE，命令行工具和其他分析工具
  • 高性能和高效内存使用
  • 可扩展：可 Python 脚本化，使用一个插件架构
  • 复用已存在的行之有效的编译器技术
  • 卓越的多线程调试支持
  • 极好的支持 C、Objective-C 和 C++
  • 可重定目标，以支持跟多的平台
  • 为调试器研究和其他创新，提供一个良好的基础
