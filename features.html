特性

LLDB 支持非常多的基础调试特性，如 DWARF 读取，支持 step，next，finish 和 backtraces 等。更有趣的特性，有：

  • 致力于可移植性和可扩展性的「插件架构」：
    • 致力于可执行文件格式的「对象文件解析」。目前支持，包括 Mach-O (32 和 64 位) 和 ELF (32 位)。
    • 致力于提取包含在一个文件中的对象文件的「对象容器解析器。当前支持 Mach-O 通用文件和 BSD 归档。
    • 调试符号介意器，以增量从对象文件获取调试信息。目前支持，包括 DWARF 和 Mach-O 符号表。
    • 符号提供者插件，用于为一个可执行对象从大量不同源收集数据。
    • 反汇编插件，为每一个架构。当前支持，包括一个针对 i386、x86-64 和 ARM/Thumb
      • 的 LLVM 反汇编器。
    • 调试器插件，实现调试要求的主机和目标特殊化的功能。
  • SWIG 生成的脚本绑定，允许 Python 访问和控制调试器库开放 API 。
  • 一个远程协议服务器，debugserver，实现 386 和 x86-64 上 Mac OS X 调试。
  • 一个命令行调试器 —— lldb 可执行程序。
  • 针对库的一个框架 API 库。
